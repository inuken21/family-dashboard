# 📝 【共働き応援】こども準備＆保育登録メモ

## システム構成・運用マニュアル（AI読み込み最適化版）

## 1. システム全体構成（Architecture）

本システムは、サーバー契約を必要とせず、すべて無料かつサーバーレスで完結する「HTML/JSフロントエンド ＋ Google Apps Script（GAS）＋ Googleスプレッドシート（DB）」の3層構造で構築されています。

```
[ ユーザーのスマートフォン (iOS/Android) ]
       │ ▲
       │ │ (① 画面表示 / WebアプリURLへアクセス)
       ▼ │
┌──────────────────────────────────────────────┐
│  フロントエンド (GitHub Pages / index.html)    │
│  - 見た目の描画 (HTML/CSS)                   │
│  - ユーザー操作の即時反映 (楽観的UI / 🎉演出)  │
│  - タッチ領域の誤爆防止ガード設計             │
└──────────────────────────────────────────────┘
       │ ▲
       │ │ (② 双方向のデータ通信：JSON形式)
       │ │ - GET : 起動時のデータ一括取得
       │ │ - POST: 操作時のピンポイント部分保存
       ▼ │
┌──────────────────────────────────────────────┐
│  バックエンド (Google Apps Script / Webアプリ) │
│  - APIサーバーとして動作 (doGet / doPost)   │
└──────────────────────────────────────────────┘
       │ ▲
       │ │ (③ データの読み書き)
       ▼ │
┌──────────────────────────────────────────────┐
│  データベース (Google スプレッドシート)        │
│  - Key-Value 形式で状態を永続化              │
└──────────────────────────────────────────────┘

```

## 2. 各層の詳細とAI向け設計仕様

### ① フロントエンド（index.html）

* **役割**: UI（ユーザーインターフェース）の提供、および状態管理。
* **AI用重要仕様（UX・バグ対策）**:
* **楽観的UI（Optimistic UI）**: チェックボックスやボタンを操作した際、GASの通信完了（1〜2秒）を待たずに、JavaScript側で「1ミリ秒」で打消し線や「🎉」エフェクトを発生させます。これにより、スマホ特有の「もっさり感」を排除しています。
* **iOS判定ガード**: iPhone等のSafariにおいて、隣接するボタンや入力欄へのタッチ暴発（誤反応）を防ぐため、要素のラッパーに `overflow: hidden;` をかけ、透明なインプットのサイズをピクセル単位（36px×34px）で厳格にクリップしています。



### ② バックエンド（GAS: Google Apps Script）

* **役割**: フロントエンドからのリクエストを受け取り、スプレッドシートへの仲介を行います。
* **エンドポイント**: `doGet(e)` で全データ取得、`doPost(e)` で単一データの保存を行います。

### ③ データベース（スプレッドシート）

* **データ構造**: AIが最も処理しやすい **Key-Value（キー・バリュー）ストア形式**を採用しています。
* A列（Key）: 一意の識別子（例：`custom_2026-06-21_kouta`, `todo_list_haruto`）
* B列 (Value): 値（テキスト、または複雑な設定・リストを格納した**JSON文字列**）

## 3. 初心者向け：ソースコード更新手順書

プログラムを変更（アップデート）する際の具体的な手順です。

### 手順A. バックエンド（GAS）の更新・デプロイ

GASのコードを変更した場合は、**「新しいバージョン」として公開（デプロイ）し直さないと、URLが同じでも中身が更新されません。**

1. **Google ドライブ** から該当の「GASプロジェクト（スクリプトエディタ）」を開きます。
2. コードを書き換えたら、画面上部の **「保存（フロッピーアイコン）」** をクリックします。
3. 画面右上にある **「デプロイ」ボタン ＞ 「デプロイの管理」** を選択します。
4. 現在のアクティブなデプロイ（Webアプリ）を選択し、右上にある **「編集（鉛筆アイコン）」** を押します。
5. **「バージョン」の項目を「新バージョン」に変更します。**（※ここが最重要です。ここを忘れると変更が反映されません）
6. 右下の **「デプロイ」** ボタンを押します。
7. 発行された「WebアプリURL」をコピーし、それがフロントエンド（index.html）の `const GAS_WEB_APP_URL` と一致しているか確認します。

### 手順B. フロントエンド（GitHub Pages）の更新

1. **GitHub** にログインし、該当のソースコード（リポジトリの `index.html`）を開きます。
2. 画面右上の **「編集（鉛筆アイコン）」** を押して、新しいソースコードを丸ごと上書き貼り付けします。
3. 画面下部または右上にある **「Commit changes...（変更をコミット）」** ボタンを押し、保存します。
4. 通常、1〜2分で自動的にインターネット上に最新版が反映されます。
* *スマホで変わらない場合の注意点*: スマホのブラウザに古いキャッシュが残っている場合があります。ブラウザのタブを完全に閉じて再開するか、プライベートブラウズモードで開いて確認してください。

## 4. 今後、新しいアプリをAIで作るための「プロンプト用要約」

次のアプリ開発の際、AI（Geminiなど）に最初に読み込ませることで、これまでの文脈を完全に理解させるためのテキストです。コピペしてそのままお使いいただけます。

```text
【新アプリ開発用の前提インプット】
私たちは現在、夫婦でリアルタイムに共有できる「子育て・家庭内タスク管理Webアプリ」を開発・運用しています。
新しいアプリ（または機能拡張）を提案・作成する際は、以下の技術スタックと設計思想を厳守してください。

■ 技術スタック
・フロントエンド: 単一のHTMLファイル（HTML5/CSS3/Vanilla JS）。外部フレームワーク（React/Vue等）は使わず、1ファイルで完結させる。
・バックエンド: Google Apps Script (GAS) をWebアプリとしてデプロイ。
・データベース: Googleスプレッドシート（A列:Key, B列:ValueのKey-Value構造）。

■ 厳守すべき設計思想
1. サーバーレス＆リアルタイム共有: データベースはGAS経由でスプレッドシートに保存し、夫婦の端末で同期する。
2. 爆速の操作感（楽観的UI）: チェックボックスのON/OFFやタスク追加時、GASの非同期通信（fetch）の完了を待たずに、JavaScript側で1ミリ秒で画面表示（打消し線、お祝いエフェクト等）を更新すること。
3. モバイルファースト・誤爆防止ガード: スマートフォン（特にiOS Safari）での片手操作を前提とする。タッチ可能なアイコン（カレンダー等）は、input要素のサイズやラッパーの「overflow: hidden」をピクセル単位で厳格に指定し、タップ判定が周囲の入力欄やボタンに1ピクセルでもはみ出して暴発しない構造にすること。
4. データの永続化: 複雑なリスト（TODOリストなど）は、フロントエンド側でJSON.stringify()を用いて文字列化し、スプレッドシートの1セルにまとめて格納・管理すること。

```
